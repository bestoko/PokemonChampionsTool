# 开发日志规范

## 目的
记录每日开发事项、完成情况、问题与待办，便于项目进度追踪与后续回顾。

## 文件命名
- 使用 `YYYY-MM-DD.md` 格式命名每日日志文件。

## 内容结构
- `日期`：日志日期
- `今日完成`
- `问题与风险`
- `明日计划`
- `备注`

## 示例
### 2026-05-19
- 今日完成
  - 初始化项目目录结构
  - 创建规范文档：`CLAUDE.md`、`docs/requirements.md`、`docs/architecture.md`
- 问题与风险
  - 需尽快确认前端技术栈和数据加载方式
- 明日计划
  - 补齐 `docs/design-guidelines.md` 和 `docs/development-plan.md`
  - 设计 `data/` JSON 数据结构
- 备注
  - MVP 先做静态单页应用，后续再扩展

# 设计规范

## 视觉风格
- 主色调：淡蓝色
- 辅助色：白色、浅灰、深蓝
- 风格：简洁、清爽、直观
- 文字：确保信息层次清晰，标题、正文、标签区分明显

## 色彩体系
- 主色：#AED7FF 或类似淡蓝
- 辅助背景：#F5FBFF、#FFFFFF
- 文字：#0F1A2B（深色主文本）、#4D637A（次要文本）
- 成功色：#23A455，警告色：#F39C12，错误色：#E74C3C

## 布局原则
- 采用卡片式信息模块
- 层次明晰，避免单页内容过于拥挤
- 保持足够空白和间距，增强阅读舒适度
- 关键操作按钮与搜索控件应易于发现

## 响应式规则
- 桌面：组件横向排列，利用宽屏空间
- 平板与移动：组件纵向堆叠，按钮与输入栏适配触控
- 最低可用视口宽度应支持 320px

## 交互规范
- 搜索与筛选：即时反馈，关键结果优先展示
- 结果卡片：要点清晰，支持“复制/保存”操作
- 用色规范：不要过度使用纯色块，主要通过边框、阴影与空白区分
- 动效：简洁轻微动画即可，避免复杂效果影响体验

## 图鉴与工具页面设计要点
- 图鉴查询：输入可快速定位宝可梦，详情页信息侧重属性、种族值、特性、招式
- 属性克制：结果展示应明确显示弱点、耐性与免疫倍率
- 速度线：排名、速度差、对比图/表需要直观
- 伤害计算器：用户输入步骤清晰，结果数值与倍率标识明晰
